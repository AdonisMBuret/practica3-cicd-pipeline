# Guía de Contribución

¡Gracias por interesarte en contribuir! Sigue estos pasos para colaborar.

## Proceso
1. **Forkea el repositorio**: Haz un fork en GitHub.
2. **Clona tu fork**: `git clone https://github.com/TU-USUARIO/practica3-cicd-pipeline.git`
3. **Crea una branch**: `git checkout -b feature/nueva-funcionalidad`
4. **Haz cambios**: Edita código, agrega pruebas.
5. **Corre pruebas localmente**: `npm test`
6. **Commitea**: `git commit -m "Descripción clara"`
7. **Pushea**: `git push origin feature/nueva-funcionalidad`
8. **Abre Pull Request**: En GitHub, describe cambios y referencia issues si aplica.

## Estándares
- Código limpio: Usa indentación consistente.
- Pruebas: Agrega cobertura para nuevos features.
- Documentación: Actualiza README si es necesario.
- No rompas el build: Verifica localmente antes de PR.

## Issues
- Reporta bugs o ideas en GitHub Issues.

¡Esperamos tu contribución!
